# SQL-Analytics-Projects Framework

## Overview

This repository demonstrates my approach to designing, documenting, and maintaining **reliable reporting metrics** and ensuring **data quality** for analytics and DBA roles. It emphasizes **accuracy, stakeholder alignment, and scalable reporting workflows**.

## Repository Contents

### 1. `reporting_metrics.sql`

SQL queries to calculate key business metrics:

* **Total Revenue:** Sum of completed orders
* **Average Order Value (AOV):** Average revenue per order
* **Monthly Active Customers (MAC):** Number of unique customers per month
* **Revenue Growth Rate:** Month-over-month revenue change
* **Repeat Customer Rate:** Proportion of customers with multiple purchases

This file demonstrates how to translate raw data into actionable insights that support business decisions.

### 2. `data_validation_checks.sql`

SQL scripts to perform **data quality and validation checks**, including:

* Row counts per day
* Date range coverage
* Negative revenue detection
* Duplicate order detection
* Null customer checks

This file shows how to ensure the underlying data is **accurate, consistent, and trustworthy** for reporting.

## Purpose

* Showcase ability to write **complex SQL queries** for reporting.
* Demonstrate **data validation and quality assurance** practices.
* Document **business logic and reporting processes** for team alignment.
* Present work clearly for **technical and non-technical stakeholders**.

## How to Use

1. Clone the repository.
2. Run SQL queries against your database or sample datasets.
3. Review results, comments, and assumptions for insights.
4. Adapt queries for similar business metrics or reports.
